# Archived phase context

These files record the state, assumptions, and handoff notes for phases 4b through 4i. They explain why parts of the scheduler, operation planner, memory policy, managed panels, montage renderer, and stage cache exist.

They are not cumulative requirements. Later ADRs and current tested behavior supersede conflicting statements. Start with [`../../current-state.md`](../../current-state.md) and consult a phase file only for historical rationale.
